sora_available {sora}R Documentation

Is SoRa available?

Description

Checks whether all of the required SoRa components are up and working and whether your API key is authorized. sora_available always returns TRUE or FALSE. sora_services returns a list of available and unavailable services. sora_key_status returns a list containing details on the status of the API key.

Usage

sora_available(services = NULL, api_key = NULL)

sora_services(api_key = NULL)

sora_key_status(api_key = NULL)

Arguments

services

If provided, checks if the specified services are available. If NULL (default), checks if any service is available.

api_key

A valid SoRa API key. Required to communicate with the SoRa API. You can request a key by filling out this form on the SoRa website.

If NULL, reads a key from the SORA_API_KEY environment variable.

If "ask" (and the session is interactive), opens a masked prompt where you can enter your key. The entered key is persistently stored in an environment variable and can be used across sessions.

Please note that you are advised not to enter your key directly using this argument. If you do, please make sure that you do not publish any code files containing the key in plain text. Failure to do so can compromise your access to the SoRa API.

Value

  • sora_available returns TRUE or FALSE

  • sora_services returns a named list of class sora_services containing information about available and unavailable services.

  • sora_key_status returns a named list of class sora_key_status.

Examples


# Checks whether the SoRa API can be accessed
sora_available()

# See what service providers are available
sora_services()

# Get details about a provided API key
sora_key_status()

# These functions can be used to create basic control flows
try(stopifnot(sora_available()))
try(stopifnot("IOER" %in% sora_services()$available))
try(stopifnot("ALLOW" %in% sora_key_status()$decision))
